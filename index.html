Document structure

Every HTML document starts with a and a root <html> element.

<!doctype html>
<html lang="en">
  <head>...</head>
  <body>...</body>
</html>

Head contents

  • <meta charset="utf-8"> — character encoding
  • <meta name="viewport" content="width=device-width, initial-scale=1"> — responsive
  • <title> — shown in browser tab
  • <link> — stylesheets, favicons, manifests
  • <script defer src="..."> — deferred script
  • <script type="module" src="..."> — ES module

Common attributes

Examples: id, class, title, hidden, data-* (custom), aria-*.

<div id="intro" class="lead" data-user-id="123" aria-hidden="false">...</div>

Accessibility (A11y)

Use semantic HTML, alt text, ARIA attributes when necessary, landmarks (<main>, <nav>, <header>, <footer>), keyboard focus management, and sufficient color contrast.

Security headers & best practices (notes)

  • Use rel="noopener noreferrer" on external target="_blank" links.
  • Prefer CSP (Content Security Policy) via headers for production.
  • Sanitize user input; do not trust client-side validation only.
